DINOv2 similarity search (FAISS)

Offline: gallery images → DINOv2 backbone → L2-normalized vectors → FAISS index. Online: query image → same backbone → ANN search → re-rank (metadata / geo) in your service layer.

UI first · embeddings & API optional

Before embeddings exist

  1. Train or freeze DINOv2; use the same resize + ImageNet normalize as predict_dino.py.
  2. Batch-encode the gallery; stack vectors (e.g. float32, N × 384 for ViT-S/14).
  3. Build FAISS (IndexFlatIP on normalized vectors, or HNSW for speed).
  4. Store parallel metadata (specimen id, species label, path, geo) for each row index.
  5. Expose a small HTTP API that accepts an image and returns top‑K neighbors + scores (see footer).
